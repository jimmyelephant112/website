    Orchestrate Data Platform

    Scheduling, deployment and monitoring of the other steps.

    Once the pipeline starts to grow, you need ways to manage all the tasks and to ensure everything remains running as expected.

    Context

    You usually add orchestration when it's proven that the project will continue to scale over time.

    Initially, you may do this in a single machine, scheduling tasks using cron. Over time, you will want to have one place to monitor what's currently running and logs of old processes.

    # Examples

    python my-pipeline-as-script.py
cron *   *   *  * my-pipeline-as-script.py
